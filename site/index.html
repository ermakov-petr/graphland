How to read these results

Results are submitted by authors or added by maintainers and reviewed for format and protocol compliance. GraphLand has no hidden test set. Unless marked “Reproduced”, results have not been independently reproduced.

Evaluation

Benchmark protocol

All tracks use public GraphLand splits. Validation performance selects the checkpoint; the corresponding public-test metric is reported.

RL

Random Low

Random stratified split with 10% train, 10% validation, and 80% test nodes. The full graph is available transductively.

split_masks_RL.csv
RH

Random High

Random stratified split with 50% train, 25% validation, and 25% test nodes. The full graph is available transductively.

split_masks_RH.csv
TH

Temporal High

Temporal split with 50% train, 25% validation, and 25% test nodes. The full graph is available transductively.

split_masks_TH.csv
THI

Temporal High · Inductive

Uses the TH temporal split. Train sees train nodes only; validation adds validation nodes; test uses the full graph snapshot.

split_masks_TH.csv · no THI file
TH and THI use the same temporal split.

They differ only in information access. In THI, validation and test nodes and their incident edges are removed from the training graph, and test nodes and their incident edges are removed from the validation graph.

Contribute

How to submit

Authors may submit their own models; maintainers may add standard open baselines.

  1. 01

    Open the public submission form.

  2. 02

    Fill in model metadata and results.

  3. 03

    A maintainer reviews the submission.

  4. 04

    A draft pull request is created.

  5. 05

    After approval and merge, the result appears on the site.

! The GitHub Issue, draft pull request, and submission discussion are public. Format and protocol review is not independent reproduction.
